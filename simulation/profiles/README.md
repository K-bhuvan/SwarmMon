# Robot / environment profiles (reference only)

These YAML files are **not loaded at runtime** today. Scenario YAML and the backend store profile **names** as strings (e.g. `generic_amr_v1`, `field_drone_v1`).

They document intended robot and site metadata for a future sim-to-real phase. See [docs/sim_to_real.md](../../docs/sim_to_real.md).
